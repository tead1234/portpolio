# 포트폴리오 웹사이트 구축/호스팅 설계안 (S3 + CloudFront)

## 1) 목표
- 프로젝트가 **지속적으로 추가**되어도 구조를 크게 바꾸지 않고 확장 가능해야 함
- 방문자가 각 프로젝트의
  - 무엇을 위해 만든 것인지(문제/목표)
  - 어떻게 구현했는지(기술/아키텍처)
  - 성과가 어땠는지(지표/결과)
  를 한눈에 파악할 수 있어야 함
- 현재 서비스 중인 프로젝트는 **바로 접근 가능한 링크** 제공

---

## 2) 추천 정보 구조(IA)

### 메인 페이지
- Hero: 소개 + 핵심 키워드
- 프로젝트 요약 카드 그리드
  - 카테고리(예: Web, Data, AI, Automation)
  - 상태(운영중, 완료, 실험)
  - 기술 스택 배지
  - 성과 요약(예: "응답속도 35% 개선")
- 필터/정렬
  - 기술 스택
  - 상태
  - 기간
  - 성과 유형(성능, 비용절감, 전환율 등)

### 프로젝트 상세 페이지
각 프로젝트를 동일한 템플릿으로 정리:
1. 문제 정의/목표
2. 역할(개인/팀, 담당 범위)
3. 구현 방식
   - 아키텍처 다이어그램(이미지)
   - 주요 기술 선택 이유
   - 핵심 트러블슈팅
4. 성과
   - 정량 지표(수치)
   - 정성 성과(피드백, 학습)
5. 링크
   - Live 서비스 URL
   - GitHub/문서/시연 영상

## 3) 콘텐츠 확장성을 위한 데이터 설계
프로젝트 추가가 잦다면, 하드코딩보다 **콘텐츠 데이터 분리**가 중요합니다.

### 권장 방식
- `projects/*.md` 또는 `projects/*.json`로 프로젝트 단위 파일 관리
- 빌드 시 목록/상세 페이지 자동 생성(SSG)

### 필수 필드 예시
```json
{
  "slug": "smart-order-dashboard",
  "title": "Smart Order Dashboard",
  "summary": "운영 대시보드 자동화로 리드타임 단축",
  "goal": "수기 보고를 제거하고 실시간 모니터링 체계 구축",
  "role": "Frontend/Backend 설계 및 배포",
  "techStack": ["Next.js", "TypeScript", "AWS Lambda", "DynamoDB"],
  "status": "live",
  "period": "2024.03 - 2024.08",
  "metrics": [
    {"label": "보고 생성 시간", "value": "2시간 → 10분"},
    {"label": "오류율", "value": "18% → 3%"}
  ],
  "liveUrl": "https://example.com",
  "repoUrl": "https://github.com/your-id/your-repo"
}
```

이렇게 구성하면 새 프로젝트는 파일 1개 추가만으로 반영할 수 있습니다.

## 4) 기술 스택 제안

### 프론트엔드
- 권장: **Next.js(Static Export)** 또는 **Astro**
- 이유: 정적 사이트 생성(SSG)에 유리, SEO/성능 확보 쉬움

### 스타일/UI
- Tailwind CSS + 컴포넌트 패턴
- 프로젝트 카드/상세 템플릿 컴포넌트화

### 분석
- GA4 또는 Plausible
- 클릭 이벤트(라이브 링크/레포 링크) 추적

---

## 5) AWS 호스팅 아키텍처 (S3 + CloudFront)

1. 정적 빌드 산출물 생성
2. S3 버킷 업로드(정적 호스팅 origin)
3. CloudFront 배포
4. Route 53 + ACM(HTTPS)
5. (선택) WAF 적용

### 운영 팁
- CloudFront 캐시 정책 분리
  - HTML: 짧은 TTL
  - JS/CSS/이미지: 긴 TTL + 파일명 해시
- SPA 라우팅/404 처리 필요 시 CloudFront custom error response 설정
- `www`/apex 도메인 리디렉션 정책 통일

---

## 6) 배포/운영 자동화(확장성 핵심)

### GitHub Actions 권장 파이프라인
- `main` 머지 시 자동 배포
  1) 빌드
  2) S3 sync
  3) CloudFront invalidation

### 브랜치 전략
- `feature/*` -> PR -> `main`
- PR마다 preview(선택)로 콘텐츠 검수 가능

### 품질 관리
- Lighthouse CI(성능/접근성/SEO)
- 링크 검사(깨진 링크 방지)
- 메타 태그/OG 이미지 검증

---

## 7) UX 관점에서 반드시 넣을 요소
- 각 프로젝트 카드에 "문제-해결-성과" 1줄 요약
- 상태 배지: `운영중`, `완료`, `PoC`
- "서비스 이용하기" 버튼은 운영중 프로젝트에서만 노출
- 상세 페이지 상단에 핵심 숫자(예: 성능, 비용, 시간 절감) 먼저 배치
- 모바일 우선 레이아웃(채용 담당자/외부인이 모바일로 보는 경우 많음)

---

## 8) 초기 구현 우선순위 (MVP)
1. 프로젝트 데이터 스키마 확정(JSON/MD)
2. 메인 리스트 + 상세 템플릿 구현
3. S3 + CloudFront 배포 연결
4. Live/Repo 링크 정책 반영
5. 기본 분석 툴 연동
